  • Real-Time Open-Domain Question Answering with Dense-Sparse Phrase Index, 2019

    • paper
    • 네이버 페이스북
      • 이 논문에서 제안하는 밀집-희소 (dense & sparse) phrase 인코딩은 phrase의 통사적, 의미적, 그리고 어휘 정보를 효과적으로 파악하며, context document의 pipeline filtering을 제거합니다. 또한 최적화 전략들을 사용하면서, 새로운 모델은 단일 4-GPU 서버에서 작동가능하며 CPU로만으로도 2TB 이하로, 600억개 까지의 phrase가 있는 위키피디아 전체를 커버 가능합니다. SQuAD-Open에 관한 이 실험은 컴퓨팅 비용을 1/6000로 줄여주며, 이는 바로, 최소 58배 빠른 end-to-end 추론 CPU에 대한 벤치마크를 의미합니다.
    • 핑퐁팀 세미나
    • repo
    • wiki: 250T -> 1.5T

  • Contextualized Sparse Representations for Real-Time Open-Domain Question Answering, 2020

    • paper
    • repo

  • summary

    • ORQA/REALM: First demonstrate that it is possible to jointly train the retriever and reader without any sparse IR components; requires novel ways of pre-training
    • DPR/RAG: It is possible to learn the dense retrieval only from Q/A pairs without pre-training!
    • DenSPI /Sparc: It is possible to index and retrieve at phrase level without requiring an explicit “reader”. Very fast inference time but slightly worse performance. Sparse features still matter.
